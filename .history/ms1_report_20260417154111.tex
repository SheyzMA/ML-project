\section{Introduction}
We study the Gaming and Mental Health dataset provided in \texttt{features.npz}. The milestone contains two supervised learning tasks: (i) regression, where the goal is to predict a continuous addiction score, and (ii) classification, where the goal is to predict addiction levels (discrete classes).

Our objective is to implement core methods from class within the provided Python skeleton, evaluate them under a consistent protocol, tune key hyperparameters, and compare final performance on held-out test data. We implemented Linear Regression (closed form), Multiclass Logistic Regression (gradient descent with softmax), and K-Nearest Neighbors (KNN) for both tasks.

\section{Method}
\subsection{Data Preparation}
Starting from \texttt{xtrain}/\texttt{xtest} and corresponding labels, we applied the same pipeline for all methods:
\begin{itemize}[noitemsep, nolistsep]
    \item \textbf{Validation split:} when not in test mode, we split the original training set into 70\% training and 30\% validation using a fixed random seed (100).
    \item \textbf{Normalization:} for each feature, we compute mean and standard deviation on the training split only, then normalize train and validation/test with these training statistics.
    \item \textbf{Bias term:} for linear and logistic regression, we prepend a column of ones to include an intercept.
\end{itemize}
For numerical safety, zero standard deviations are replaced by 1.

\subsection{Linear Regression}
We use the closed-form least-squares solution. Given design matrix $\mathbf{X}$ and targets $\mathbf{y}$,
\[
\mathbf{w} = (\mathbf{X}^{\top}\mathbf{X})^{-1}\mathbf{X}^{\top}\mathbf{y}.
\]
In implementation, we use the pseudo-inverse form $\mathbf{w}=\mathbf{X}^{\dagger}\mathbf{y}$ for better numerical robustness. Prediction is $\hat{\mathbf{y}}=\mathbf{X}\mathbf{w}$.

\subsection{Logistic Regression}
For multiclass classification, we optimize softmax regression with full-batch gradient descent. Let $\mathbf{W}\in\mathbb{R}^{D\times C}$ and logits $\mathbf{Z}=\mathbf{XW}$. Probabilities are
\[
\mathrm{softmax}(\mathbf{Z})_{i,c}=\frac{e^{Z_{i,c}}}{\sum_{c'}e^{Z_{i,c'}}}.
\]
We use one-hot targets and update
\[
\mathbf{W} \leftarrow \mathbf{W} - \eta\,\frac{1}{N}\mathbf{X}^{\top}(\mathbf{P}-\mathbf{Y}).
\]
To avoid overflow in exponentials, logits are shifted by row-wise maxima before exponentiation.

\subsection{K-Nearest Neighbors (KNN)}
KNN stores training samples and predicts by nearest neighbors under Euclidean distance.
\begin{itemize}[noitemsep, nolistsep]
    \item \textbf{Classification:} majority vote among $k$ nearest labels.
    \item \textbf{Regression:} average of neighbor target values.
\end{itemize}
We tune $k$ on the validation split and retrain on full training data before test evaluation.

\section{Experiments and Results}
\subsection{Setup and Hyperparameter Search}
Metrics are MSE for regression, and Accuracy/Macro-F1 for classification. We searched:
\begin{itemize}[noitemsep, nolistsep]
    \item \textbf{KNN:} $k\in\{1,3,5,7,9,11,15,21\}$.
    \item \textbf{Logistic Regression:} learning rate $\eta\in\{10^{-4},3\cdot10^{-4},10^{-3},3\cdot10^{-3},10^{-2},3\cdot10^{-2}\}$ and iterations $\in\{200,500,1000,2000\}$.
\end{itemize}
Best validation settings were: KNN-regression $k=9$, KNN-classification $k=9$, Logistic Regression $(\eta=0.01,\;\text{iters}=1000)$.

\section{Discussion and Conclusion}
For regression, Linear Regression clearly outperformed KNN on both validation and test MSE, suggesting that a mostly linear relationship is already captured well after normalization and bias augmentation.

For classification, Logistic Regression achieved the best accuracy (84.50\% on test), while KNN reached lower accuracy but slightly better test macro-F1, indicating a different class-balance behavior. KNN with small $k$ overfit strongly (perfect training accuracy at $k=1$), while moderate $k$ improved generalization.

A practical challenge in logistic training is numerical stability in the softmax computation. Using shifted logits prevented overflow/\texttt{NaN} behavior and enabled stable convergence for larger learning rates.

Overall, Milestone 1 validates the end-to-end pipeline and establishes competitive baselines. For Milestone 2, promising extensions include regularized models, stronger feature engineering, and class-imbalance-aware optimization/objectives.
